# higgsfield-fx Tuner — 피드백 누적·큐레이션 메타 스킬

`higgsfield-fx` 스킬을 **사용 패턴에 맞춰 자동으로 진화**시키는 메타 스킬. 매 피드백마다 SKILL.md를 건드리면 무거워지므로, 가벼운 누적 → 임계치 도달 시 큐레이션 → 패턴 있는 것만 반영하는 두 단계 구조로 동작한다.

**모든 출력 메시지·미리보기·확인 질문은 한국어로 한다.** (영문 어휘는 SKILL.md 본문 어휘 표에서만 사용)

## 작업 디렉토리

이 스킬은 다음 경로의 파일을 자동 관리한다 (없으면 첫 동작 시 생성):

```
~/.claude/skills/higgsfield-fx/
├── SKILL.md             # 본 스킬 (큐레이션 결과로 직접 수정)
├── CHANGELOG.md         # 사람이 읽는 변경 이력 (보고서 출력의 원본)
├── feedback-log.md      # raw 피드백 한 줄씩 누적 (분석 X)
└── .backup/
    └── SKILL.md.YYYYMMDD-HHMMSS  # 자동 백업 (최근 5개만 유지)
```

작업 전에 항상 이 경로가 존재하는지 `Bash`로 확인하고, 없으면 생성한다.

## 4가지 모드

| 모드 | 트리거 | 동작 |
| --- | --- | --- |
| **capture** | 자연어 피드백 자동 발동 | `feedback-log.md`에 한 줄 추가, 한국어로 한 줄 확인 응답 |
| **review** | `/higgsfield-fx-tuner review` 또는 임계치 7개 도달 후 사용자 동의 | 누적 피드백 분석 → 변경안 미리보기 → 사용자 승인 → SKILL.md & CHANGELOG.md 일괄 업데이트 |
| **report** | `/higgsfield-fx-tuner report` | CHANGELOG.md 기반 보고서 텍스트 코드 블록 출력 |
| **status** | `/higgsfield-fx-tuner status` | 현재 상태 4~5줄 요약 |

명시 명령(`review` / `report` / `status`)이 입력에 포함되면 그 모드로, 그 외 자연어 피드백 톤이면 capture로 분기.

---

## capture 모드 — 자동 누적 (가벼움 유지)

### 발동 조건
사용자가 다음과 같은 자연어 피드백을 줄 때:
- 평가형: "별로야", "너무 강해/약해", "이거 좋아"
- 추가 요청: "이런 어구 추가해줘", "○○도 넣을 수 있어?"
- 제거 요청: "이 어구는 빼줘", "이건 안 쓸래"
- 톤 조정: "더 어둡게", "차분하게"

### 절차

1. **카테고리 자동 추정** — 피드백의 키워드를 7카테고리(Impact / Particle / Fluid / Optical / Atmospheric / Magical / Material)에 매핑. 애매하면 `category=unknown`.
2. **`feedback-log.md`에 append** — 다음 한 줄 형식:
   ```
   YYYY-MM-DD | category=<name> | "<원문>" | context: <짧은 맥락>
   ```
   - 파일이 없으면 헤더 (`# Feedback Log\n\n`) 포함하여 생성
   - 처리되지 않은 피드백은 파일 상단의 `## 미처리` 섹션에, 큐레이션 끝난 항목은 `## 처리됨 (YYYY-MM-DD)` 섹션에 보관
3. **사용자에게 한국어 한 줄 응답**:
   ```
   📝 피드백 기록됐어요 (3/7)
   ```
   - `(N/7)` 형식으로 누적 카운트 표시
4. **임계치 7개 도달** — 다음번 capture 발동 시 한 번만 부드럽게 알림:
   ```
   📝 피드백 기록됐어요 (7/7)
   
   💡 7개 모였어요. `/higgsfield-fx-tuner review` 입력하시면 패턴을 정리해서 스킬에 반영해드려요. (지금 안 해도 괜찮아요)
   ```
   - 알림 표시 후 `feedback-log.md`에 `<!-- threshold-notified: 7 -->` 같은 마커 추가 → 다음 capture에선 알림 없음 (review 끝나면 마커 제거)

### capture가 하지 않는 일
- 분석·패턴 추출 X (review에서 일괄 처리)
- 다른 피드백과의 중복 검사 X
- SKILL.md 수정 X

## review 모드 — 큐레이션·반영

### 1단계: 분석

`feedback-log.md`의 `## 미처리` 섹션 전체를 읽어 다음으로 분류:

| 분류 | 기준 | 처리 |
| --- | --- | --- |
| **반복 패턴** | 동일·유사 의견 3회 이상 (의미 단위로) | 룰화 후보 |
| **신규 어휘** | 사용자가 명시한 영문/한글 어휘·구절 (3회 이상 등장) | 어휘 추가 후보 |
| **일회성** | 1~2회만 등장한 의견 | 자동 폐기 후보 |
| **충돌** | 같은 카테고리에 반대·모순된 의견 | 사용자에게 한 줄 질문 |

### 2단계: 변경안 미리보기 (한국어)

다음 형식의 코드 블록 1개로 출력:

```
📋 N개 피드백 분석 결과

## 반영 추천 (3개)
1. [룰] Magical 카테고리에 항상 "subtle/faint" 강도 어휘 동반
   근거: 5회 반복 (5/3, 5/5, 5/7, 5/9, 5/12)
2. [어휘] Particle 카테고리 표에 "smoldering ember trail" 추가
   근거: 3회 등장
3. [어휘] Material 카테고리 표에 "molten edge" 추가
   근거: 3회 등장

## 폐기 (3개) — 일회성 의견이라 일반화 어려움
- "이 컷에선 더 어둡게" (특정 컷 한정)
- (다른 항목 요약)
- (다른 항목 요약)

## 확인 필요 (1개) — 충돌
- 5/4: "Glow는 강하게" / 5/8: "Glow는 약하게"
  → 어떤 톤을 디폴트로 할까요? (강하게 / 약하게 / 둘 다 보존)

반영할까요? (예 / 아니오 / 번호로 선택 - 예: "1,2번만")
```

### 3단계: 사용자 승인 후 적용

1. **백업 생성** — `~/.claude/skills/higgsfield-fx/.backup/SKILL.md.YYYYMMDD-HHMMSS`로 SKILL.md 복사
2. **백업 정리** — `.backup/` 안에 6개 이상이면 가장 오래된 것 삭제 (최근 5개만 유지)
3. **SKILL.md 수정**:
   - **어휘 추가**: 해당 카테고리의 "핵심 동사·명사" 표 끝에 추가
   - **룰 추가**: "베스트 프랙티스 룰" 섹션의 적절한 위치에 한 줄 불릿으로 추가
   - **클리셰 구절 추가**: 해당 카테고리의 "클리셰 구절" 목록에 톤 라벨과 함께 추가
4. **CHANGELOG.md 갱신** — 파일 상단(역시간순)에 항목 추가:
   ```
   ## 2026-05-12 — 친구 패턴 기반 진화 #N

   ### 추가된 룰
   - Magical 카테고리에 항상 "subtle/faint" 강도 어휘 동반
     근거: 5회 반복 패턴

   ### 추가된 어휘
   - Particle: smoldering ember trail (3회 등장)
   - Material: molten edge (3회 등장)

   ### 폐기 (3개)
   일회성 의견이라 일반화 어려움 (자세한 내용은 feedback-log.md 처리됨 섹션 참고)

   ### 미해결
   - (충돌 항목이 보류된 경우 여기에)
   ```
   - 파일이 없으면 `# Changelog\n\n사람이 읽는 변경 이력. 보고서 출력의 원본.\n\n` 헤더와 함께 생성
5. **feedback-log.md의 처리된 항목 이동** — `## 미처리`에서 빼서 `## 처리됨 (YYYY-MM-DD)` 섹션으로. 폐기된 항목도 거기로 (지우지 않음, 추적용)
6. **threshold-notified 마커 제거** — `feedback-log.md`의 마커 주석 삭제 (다음 7개 사이클 시작)
7. **사용자에게 확인 한 줄**:
   ```
   ✅ 반영 완료. 지금까지 N번 진화했어요.
   ```

### 4단계: 사이즈 가드레일

SKILL.md 줄 수가 **300줄 이상**이면 review 끝에 한 줄 추가:
```
🔍 SKILL.md가 N줄로 길어졌어요. 중복 어휘 통합·압축할까요? (예 / 나중에)
```

사용자 OK 시:
- 같은 카테고리에서 의미 중복 어휘 통합
- 룰 중에서 다른 룰의 부분집합인 것 흡수
- 결과를 미리보기로 보여주고 다시 승인 받음
- 통과 시 SKILL.md 갱신 + CHANGELOG에 "압축 #N" 항목 추가

## report 모드 — 사용자에게 보낼 보고서

`CHANGELOG.md`를 읽어 다음 형식의 **코드 블록 1개**로 출력. 친구는 그걸 그대로 카톡/이메일로 복붙.

```
📋 higgsfield-fx 진화 보고서
기간: YYYY-MM-DD ~ YYYY-MM-DD (CHANGELOG.md의 첫/마지막 항목 날짜)

이 기간에 N번 큐레이션, M개 패턴 반영, K개 폐기.

## 추가된 룰
1. <룰 한 줄> (YYYY-MM-DD)
   근거: <근거 한 줄>
2. ...

## 추가된 어휘
- <카테고리>: <어휘> (N회 등장)
- ...

## 폐기된 일회성 의견
N개 (대부분 단일 컷 한정 의견)

## 미해결 충돌
- (있으면 여기에, 없으면 "없음")
```

추가 설명 없이 코드 블록만 출력. 출력 위에 한 줄 안내:
```
아래 보고서를 카톡/이메일로 복사해서 보내세요.
```

## status 모드 — 빠른 현황

다음 형식으로 4~6줄 출력:

```
📊 higgsfield-fx 현황
- 누적 피드백: 4/7 (미처리)
- 마지막 큐레이션: 2026-05-12 (3일 전)
- 마지막 변경: Magical 카테고리에 강도 어휘 룰 추가
- SKILL.md 사이즈: 245줄 (압축 임계치 300줄)
- 백업: 4개 (가장 최근: SKILL.md.20260512-143205)
```

각 항목은 다음에서 추출:
- 누적 피드백: `feedback-log.md`의 `## 미처리` 섹션 줄 수
- 마지막 큐레이션 날짜: `CHANGELOG.md` 첫 항목 날짜
- 마지막 변경: `CHANGELOG.md` 첫 항목의 첫 룰/어휘 한 줄
- SKILL.md 사이즈: `wc -l`
- 백업: `ls .backup/ | wc -l` 및 가장 최근 파일명

## 베스트 프랙티스 룰 (메타 스킬 내부)

- **capture는 가벼움 유지** — 한 줄 append만, 분석·중복검사 절대 X (토큰 절약)
- **임계치 알림 1회만** — `<!-- threshold-notified: 7 -->` 마커로 중복 알림 방지
- **일회성은 자동 폐기** — 친구한테 매번 묻지 말고, review에서 일괄 보고
- **충돌만 한 줄 질문** — 모순되는 피드백 한해
- **백업 항상** — SKILL.md 수정 직전 자동, 최근 5개만 유지
- **사이즈 가드** — SKILL.md ≥300줄에서 압축 제안
- **한국어 인터랙션** — 출력·미리보기·확인 메시지 모두 한국어
- **베이스/특화 표시** — CHANGELOG.md 항목 제목에 "친구 패턴 기반 진화 #N"이라고 명시

## 안전 장치

- **롤백 지원** — 사용자가 "방금 변경 되돌려줘", "원래대로", "롤백해줘" 같은 자연어 요청 시:
  1. `.backup/`에서 가장 최근 파일을 확인
  2. 미리보기: "마지막 변경: <CHANGELOG 첫 항목 요약>. 이걸 되돌릴까요? (예/아니오)"
  3. 사용자 OK → SKILL.md를 백업으로 덮어쓰고, CHANGELOG.md 첫 항목에 `[롤백됨 YYYY-MM-DD]` 표시
- **파일 손상 감지** — feedback-log.md / CHANGELOG.md 형식이 깨졌을 때 추측해서 수정하지 말고 사용자에게 통보 후 중단
- **Bash로 cat 금지** — 파일 분석은 `Read` 도구 사용

## 주의사항

- **첫 동작 시** 작업 디렉토리에 `feedback-log.md`, `CHANGELOG.md`, `.backup/`이 없으면 자동 생성 (한국어 헤더 포함)
- **higgsfield-fx 스킬이 설치되어 있지 않으면** 사용자에게 통보 후 중단 (`~/.claude/skills/higgsfield-fx/SKILL.md` 부재 시)
- **review 미리보기에서 사용자가 "1,3번만" 같은 부분 선택**을 하면 그 항목만 반영, 나머지는 미처리로 다시 보존
- **사용자가 review 도중 취소** ("아니오" / "다음에")하면 어떤 파일도 수정하지 않고 종료
- **카테고리 자동 매핑이 애매**한 피드백은 `category=unknown`으로 기록하고, review에서 사용자에게 카테고리를 묻는다
